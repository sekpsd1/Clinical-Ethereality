# ข้อความส่งลูกค้า: ขอรีวิว SOP ร้านยา/ใบสั่งยา

เอกสารนี้เป็นข้อความร่างสำหรับ owner ส่งให้ลูกค้าทาง LINE หรืออีเมล เพื่อขอให้ลูกค้าตรวจ workflow การใช้คลินิกเป็นร้านยาและการ fulfill ยาตามใบสั่งแพทย์ ห้ามใส่เลขใบอนุญาต เลขบัญชี เลข PromptPay หรือข้อมูลส่วนบุคคลจริงลงในไฟล์นี้

## ข้อความสั้นสำหรับส่งใน LINE

สวัสดีครับ ทีมงานร่าง flow/SOP สำหรับการใช้คลินิกเป็นร้านยาและการจัดการใบสั่งยาเบื้องต้นให้ตรวจแล้วครับ

รบกวนช่วยตรวจ 3 ส่วนหลัก:

1. การตรวจใบสั่งยาโดยเภสัชกร
2. การเตรียมยาและการห้ามเตรียมก่อน payment/prescription ผ่าน
3. การแพ็กและจัดส่งผ่าน Thailand Post EMS พร้อม tracking number

มีคำถามที่ต้องยืนยันเพิ่มเติม:

- เลขใบอนุญาตร้านยา/คลินิกสำหรับจ่ายยาใช้รายการใด
- ชื่อเภสัชกรและเลขใบอนุญาตเภสัชกรเป็นข้อมูลใด
- มีสินค้าควบคุม สินค้าแช่เย็น หรือสินค้าที่ต้องเก็บพิเศษไหม
- อนุญาตให้เปลี่ยนสินค้า/ยาแทนกันได้ไหม ถ้าได้ ใครอนุมัติ
- ถ้าเภสัชกรจะปฏิเสธใบสั่งยา ต้องให้ติดต่อแพทย์ก่อนทุกครั้งไหม
- กรณีจัดส่งไม่สำเร็จ ที่อยู่ผิด พัสดุหาย หรือตีกลับ ต้องส่งใหม่หรือคืนเงินอย่างไร
- ต้องมี prescription PDF, ฉลากยา หรือเอกสารแนบรูปแบบเฉพาะไหม
- มี SOP ภายในของคลินิก/ร้านยาที่ต้องใส่เพิ่มไหม

ถ้าลูกค้ายืนยันตามนี้ ทีมงานจะใช้เป็น baseline สำหรับ workflow ในระบบต่อไปครับ

## ข้อความละเอียดสำหรับส่งในอีเมล

สวัสดีครับ

ทีมงานได้จัดทำร่าง SOP สำหรับ pharmacy/prescription fulfillment flow เพื่อให้ลูกค้าตรวจและยืนยันก่อนนำไปใช้เป็น baseline ในระบบ Clinical Ethereality แล้วครับ

ขอบเขตของร่างนี้ครอบคลุม:

- การออก prescription note หลัง consult
- การตรวจใบสั่งยาโดยเภสัชกร
- เงื่อนไขการอนุมัติ/ปฏิเสธ prescription
- การสั่งซื้อสินค้าที่ต้องใช้ใบสั่งยา
- การตรวจ payment/slip ก่อนเริ่มเตรียมยา
- การเตรียมยา แพ็กสินค้า และจัดส่งผ่าน Thailand Post EMS
- การบันทึก tracking number
- กรณีผิดปกติ เช่น payment rejected, stock ไม่พอ, ข้อมูลผู้ป่วยไม่ตรง, พัสดุเสียหาย/สูญหาย/ตีกลับ
- audit และ privacy สำหรับข้อมูลผู้ป่วย ใบสั่งยา payment และ shipment

รบกวนลูกค้าช่วยตรวจและตอบกลับในหัวข้อต่อไปนี้:

1. ข้อมูลใบอนุญาตร้านยา/คลินิกสำหรับจ่ายยา
2. ชื่อเภสัชกรและเลขใบอนุญาตเภสัชกร
3. มีสินค้าควบคุม สินค้าแช่เย็น หรือสินค้าที่ต้องจัดเก็บพิเศษหรือไม่
4. การเปลี่ยนสินค้า/ยาแทนกันอนุญาตหรือไม่ และใครเป็นผู้อนุมัติ
5. หากเภสัชกรจะปฏิเสธ prescription ต้องติดต่อแพทย์ก่อนหรือไม่
6. นโยบายกรณีจัดส่งไม่สำเร็จ ที่อยู่ผิด พัสดุสูญหาย พัสดุตีกลับ การส่งใหม่ และการคืนเงิน
7. ต้องมี prescription PDF, ฉลากยา หรือเอกสารแนบรูปแบบเฉพาะหรือไม่
8. มี SOP ภายในของคลินิก/ร้านยาที่ต้องเพิ่มเติมจากร่างนี้หรือไม่

หมายเหตุ: ไม่ควรส่งเลขบัญชี API key หรือข้อมูลลับผ่านช่องทางที่ไม่ปลอดภัย หากมีข้อมูล sensitive ให้ส่งผ่านช่องทางที่ตกลงกันไว้เท่านั้นครับ
